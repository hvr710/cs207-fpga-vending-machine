# 售货机销售控制模块 (sales_mode) 技术说明文档

## 1. 模块综述

`sales_mode` 模块负责系统的普通顾客购买流程。该模块采用有限状态机（FSM）设计，完成饮料翻页选择、库存/状态/价格检查、投币付款、出货等待、取货确认、超时退款、无操作自动返回销售主页以及 LED/数码管显示数据输出。

本版本固定支持 **4 种饮料**：COLA、SODA、TEA、H2O。为了适配 EGO1 板子的 8 位七段数码管，选择界面采用一页显示一种饮料的方式，显示布局为：

```text
[name4][price2][S][stock]
```

示例：`COLA04S5` 表示 COLA，价格 4，库存 5。价格统一控制在 **10 以内**，方便上板演示时直接用拨码开关表示金额，不需要临时换算复杂二进制数。

## 2. 接口定义

### 2.1 基础与输入信号

| 信号名 | 位宽 | 类型 | 说明 |
|---|---:|---|---|
| `clk` | 1 | Input | 系统主时钟，默认 100 MHz |
| `rst_n` | 1 | Input | 异步复位，低有效 |
| `sales_en` | 1 | Input | 销售模式总使能，拉高后进入销售主页 |
| `switch_in` | [7:0] | Input | EGO1 底部 8 个拨码开关，付款阶段作为金额输入 |

### 2.2 交互信号（按键）

本模块假设所有按键输入已经由顶层完成消抖，并转换为高有效单周期脉冲。

| 信号名 | 位宽 | 类型 | EGO1 建议按键 | 说明 |
|---|---:|---|---|---|
| `btn_confirm` | 1 | Input | S0 | 确认选择、确认付款、确认取货 |
| `btn_prev` | 1 | Input | S1 | 上一页饮料 |
| `btn_next` | 1 | Input | S2 | 下一页饮料 |
| `btn_pay` | 1 | Input | S3 | 将当前 `switch_in[7:0]` 加入余额 |
| `btn_cancel` | 1 | Input | S4 | 取消订单 / 返回 |

### 2.3 商品数据输入

商品价格和库存不在销售模块内永久存储，建议由顶层或中央寄存器堆提供。销售模块只读取当前数据并在交易完成时输出写脉冲。

| 信号名 | 位宽 | 类型 | 说明 |
|---|---:|---|---|
| `price0~price3` | [7:0] | Input | 4 种饮料价格，建议演示版均不超过 10 |
| `stock0~stock3` | [3:0] | Input | 4 种饮料库存，建议 0~9 方便数码管显示 |
| `enabled_mask` | [3:0] | Input | 是否在售，1=在售，0=停售 |

推荐初始商品表：

| 编号 | 名称 | 数码管显示 | 价格 | 库存 | 演示金额拨码 |
|---:|---|---|---:|---:|---|
| 0 | COLA | `COLA04S5` | 4 | 5 | `00000100` |
| 1 | SODA | `SOdA05S6` | 5 | 6 | `00000101` |
| 2 | TEA | `tEA 03S8` | 3 | 8 | `00000011` |
| 3 | H2O | `H2O 02S9` | 2 | 9 | `00000010` |

### 2.4 输出与寄存器交互

| 信号名 | 位宽 | 类型 | 说明 |
|---|---:|---|---|
| `drink_id` | [1:0] | Output | 当前选中的饮料编号 0~3 |
| `sale_we` | 1 | Output | 取货成功时拉高 1 个周期，用于扣库存和增加总收入 |
| `sale_idx` | [1:0] | Output | 本次成功销售的饮料编号 |
| `sale_amount` | [7:0] | Output | 本次应加入总收入的金额，一般等于当前价格 |
| `refund_pulse` | 1 | Output | 取消/超时时拉高 1 个周期，用于触发退款提示 |
| `refund_amount` | [7:0] | Output | 本次退款金额 |
| `exit_to_main` | 1 | Output | 在销售主页按取消时拉高 1 个周期，可选接到主状态机 |

### 2.5 显示与调试输出

| 信号名 | 位宽 | 类型 | 说明 |
|---|---:|---|---|
| `paid_amount` | [7:0] | Output | 当前已投金额 |
| `current_price` | [7:0] | Output | 当前饮料价格 |
| `current_stock` | [3:0] | Output | 当前饮料库存 |
| `led_out` | [15:0] | Output | 16 个 LED 输出，支付进度条/流水灯/错误提示 |
| `view_data` | [39:0] | Output | 8 位数码管字符数据，每位 5 bit，顺序为 digit7 到 digit0 |
| `state_code` | [3:0] | Output | 当前销售状态码 |
| `error_code` | [3:0] | Output | 当前错误码 |
| `countdown_sec` | [3:0] | Output | 取货等待倒计时，显示 5 到 0 |

## 3. 核心逻辑说明

### 3.1 状态机设计 (FSM)

模块内部包含 9 个状态：

| 状态名 | 编码 | 说明 |
|---|---:|---|
| `S_IDLE` | 0 | 等待进入销售模式 |
| `S_SELECT` | 1 | 销售主页/饮料选择页 |
| `S_CHECK` | 2 | 检查饮料是否在售、库存是否充足、价格是否有效 |
| `S_PAY` | 3 | 支付阶段，按 `btn_pay` 累加金额 |
| `S_DISPENSE` | 4 | 出货过渡状态 |
| `S_WAIT_TAKE` | 5 | 等待顾客取货，5 秒倒计时 |
| `S_COMPLETE` | 6 | 交易完成提示 |
| `S_REFUND` | 7 | 退款提示 |
| `S_ERROR` | 8 | 错误提示 |

主流程：

```text
S_IDLE -> S_SELECT -> S_CHECK -> S_PAY -> S_DISPENSE -> S_WAIT_TAKE
                                   |                         |
                                   |                         |-- S_COMPLETE 取货成功
                                   |                         |-- S_REFUND 超时/取消
                                   |-- S_ERROR 异常
```

### 3.2 翻页选择逻辑

`S_SELECT` 是销售模式的主页面。每种饮料占一页：

- `btn_next`：切换到下一种饮料，3 号之后回到 0 号。
- `btn_prev`：切换到上一种饮料，0 号之前回到 3 号。
- `btn_confirm`：确认当前页饮料并进入检查状态。
- `btn_cancel`：在销售主页退出，可通过 `exit_to_main` 通知顶层返回主菜单。

数码管显示格式为：

```text
[name4][price2][S][stock]
```

例如：

```text
COLA04S5
SOdA05S6
tEA 03S8
H2O 02S9
```

### 3.3 支付逻辑

进入 `S_PAY` 后，顾客通过拨码开关输入本次投币金额：

1. 将 `switch_in[7:0]` 拨成金额。
2. 按 `btn_pay`，模块将该金额累加到 `paid_amount`。
3. 按 `btn_confirm` 确认付款。
4. 若 `paid_amount >= current_price`，进入出货流程；否则保持在支付状态并置 `ERR_NOT_ENOUGH`。

支付界面数码管显示：

```text
bALxxPyy
```

其中 `xx` 是当前余额，`yy` 是当前价格。比如 COLA 的价格为 4，则可显示为 `bAL04P04`。LED 同时显示 16 位支付进度条。

### 3.4 取货与退款逻辑

付款成功后进入 `S_WAIT_TAKE`，数码管显示：

```text
PUSH   5
```

倒计时从 5 到 0。此处 `PUSH` 表示顾客需要按确认键 S0 完成取货。

- 5 秒内按 `btn_confirm`：输出 `sale_we=1` 一个周期，进入 `S_COMPLETE`。
- 5 秒内按 `btn_cancel`：输出 `refund_pulse=1`，进入 `S_REFUND`。
- 5 秒内无操作：自动退款，输出 `refund_pulse=1`，进入 `S_REFUND`。

本模块的设计重点是：**只有顾客确认取货后才计入销售额**，付款后但未取货不会计入收入。

### 3.5 无操作自动返回

为了避免顾客停留在某个页面不操作，本模块加入了无操作超时逻辑：

- 在 `S_PAY` 状态，如果超过 `INACTIVITY_TIMEOUT_CYCLES` 没有任何按键操作，则自动退款并返回销售流程。
- 在 `S_ERROR`、`S_COMPLETE`、`S_REFUND` 等提示状态，如果超过 `MESSAGE_TIMEOUT_CYCLES` 无操作，则自动回到 `S_SELECT` 销售主页。
- `S_SELECT` 本身就是销售主页，不再额外超时退出。

默认设置：

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `TAKE_TIMEOUT_CYCLES` | 500,000,000 | 5 秒取货超时 |
| `INACTIVITY_TIMEOUT_CYCLES` | 3,000,000,000 | 30 秒无操作超时 |
| `MESSAGE_TIMEOUT_CYCLES` | 300,000,000 | 3 秒提示页自动返回 |

## 4. EGO1 板上操作流程

### 4.1 正常购买

1. 进入销售模式后，数码管显示第 0 种饮料，例如 `COLA04S5`。
2. 按 S1/S2 上下翻页，选择饮料。
3. 按 S0 确认选择。
4. 用 SW0~SW7 拨出投币金额。演示时可直接用 2、3、4、5 这些小数值。
5. 按 S3 将金额加入余额，可重复加钱。
6. 按 S0 确认付款。
7. 出货后，5 秒内按 S0 确认取货。
8. 模块输出 `sale_we=1`，顶层/寄存器堆扣库存并增加收入。

### 4.2 异常操作

| 场景 | 操作方式 | 结果 |
|---|---|---|
| 余额不足 | `paid_amount < price` 后按 S0 | 保持在支付状态，显示错误码 3 |
| 付款阶段无操作 | 支付阶段长时间不按键 | 自动退款，返回销售主页 |
| 出货后未取货 | 5 秒内不按 S0 | 自动退款，不计入收入 |
| 饮料停售 | `enabled_mask = 0` | 进入错误状态，错误码 1 |
| 库存不足 | 选择库存为 0 的饮料 | 进入错误状态，错误码 2 |
| 价格异常 | 当前价格为 0 | 进入错误状态，错误码 4 |

## 5. 集成建议 [To Member C]

1. **按键消抖**：本模块假设 `btn_confirm/btn_prev/btn_next/btn_pay/btn_cancel` 已经是消抖后的单周期脉冲。
2. **商品数据**：建议由 C 的寄存器堆提供 `price0~price3`、`stock0~stock3`、`enabled_mask`。销售模块不永久保存库存和价格。
3. **写回逻辑**：当 `sale_we` 拉高时，寄存器堆应执行：`stock[sale_idx]--`，`total_revenue += sale_amount`。
4. **退款逻辑**：当 `refund_pulse` 拉高时，只显示退款提示或记录退款事件；由于本模块只有取货成功后才拉高 `sale_we`，所以超时退款时不需要回滚收入。
5. **数码管显示**：`view_data[39:0]` 使用 8 个 5-bit 字符 ID。C 需要在 `seg7_driver` 或 `display_mux` 中建立字符 ID 到七段码的映射。
6. **LED**：`led_out[15:0]` 可直接接入 LED driver。支付阶段是进度条，取货阶段是变频流水灯，错误/完成/退款有固定图案。
7. **退出机制**：如果顶层需要从销售模式回主菜单，可以把 `exit_to_main` 接到主状态机；如果不需要，可以忽略该输出。
